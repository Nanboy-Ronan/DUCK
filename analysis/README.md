# DUCX Analysis

This directory contains the analysis code for **DUCX: Decomposing Unfairness in Tool-Using Chest X-ray Agents**.

The paper audits MedRAX-style chest X-ray agents at two levels:

- **End-to-end fairness**: final multiple-choice QA performance across demographic groups.
- **Process-level fairness decomposition**: intermediate disparities in tool exposure, tool transitions, and model reasoning.

The main entry point is [fairness_posthoc.py](fairness_posthoc.py). It reads agent trajectory logs, joins question-level and case-level metadata, extracts tool paths and final answers, and writes the tables used to inspect DUCX fairness behavior.

## Paper Metrics

DUCX reports end-to-end performance with:

- `ACC`: overall accuracy.
- `Delta-ACC`: absolute subgroup accuracy gap.
- `DP`: demographic parity gap over predicted answer distributions.
- `EoD`: equalized-odds gap over answer classes.
- `FUT`: fairness-utility tradeoff, combining subgroup utility and disparity.

These metrics are implemented in [compute_fairness_metrics.py](compute_fairness_metrics.py). The script expects `per_question_features.csv` outputs generated by `fairness_posthoc.py`.

## Decomposition Lenses

The paper localizes bias into three agent-specific sources.

### 1. Tool Exposure Bias

Question: when a tool appears in an agent trajectory, does its utility differ across demographic groups?

Primary outputs:

- `lens_agentic_conditional_tool_utility.csv`
- `lens_agentic_conditional_tool_utility_gap.csv`

Supporting output:

- `lens_agentic_tool_selection_bias.csv`: subgroup differences in whether each tool is used.

### 2. Tool Transition Bias

Question: does the agent route different groups through different tool paths?

Primary outputs:

- `lens_agentic_transition_rates_by_group.csv`
- `lens_agentic_transition_divergence.csv`

The transition analysis treats each trajectory as a sequence of tool states and compares subgroup transition distributions, corresponding to the paper's tool-routing view.

### 3. Model Reasoning Bias

Question: after tools return results, does the final synthesis differ across demographic groups?

Primary outputs:

- `lens_llm_reasoning_bias.csv`
- `lens_llm_reasoning_bias_by_group.csv`

The paper emphasizes differences in reasoning quality, hedging / uncertainty expression, and demographic framing. `--enable-llm-judge` adds a judge-based `reasoning_quality_score`; lexical features such as hedging and demographic-term counts are extracted directly from final responses.

## Typical Commands

ChestAgentBench / EuroRAD:

```bash
python analysis/fairness_posthoc.py \
  --log-path logs/qwen3vl8b-vllm/<run_log>.json \
  --meta-q-path data/chestagentbench/metadata.jsonl \
  --meta-case-path data/eurorad_metadata.json \
  --out-dir logs/chexagentbench/analysis/qwen3vl8b-vllm/fairness_posthoc \
  --enable-llm-judge
```

MIMIC-FairnessVQA:

```bash
python analysis/fairness_posthoc.py \
  --log-path logs/qwen3vl8b-vllm-mimic/<run_log>.json \
  --meta-q-path data/mimic/medrax_input_all_2000.jsonl \
  --meta-case-path data/mimic/mimic_sample_400.csv \
  --out-dir logs/mimic/analysis/qwen3vl8b-vllm/fairness_posthoc \
  --enable-llm-judge
```

Batch mode over baseline subdirectories:

```bash
python analysis/fairness_posthoc.py \
  --input-root logs/chexagentbench \
  --out-root logs/chexagentbench/analysis \
  --meta-q-path data/chestagentbench/metadata.jsonl \
  --meta-case-path data/eurorad_metadata.json \
  --enable-llm-judge
```

Full paper-figure reproduction:

```bash
python analysis/generate_paper_figures.py \
  --chex-analysis-root logs/chexagentbench/analysis \
  --mimic-analysis-root logs/mimic/analysis/fairness \
  --chex-tool-present-csv logs/chexagentbench/analysis/tool_present_subgroup_metrics_only.csv \
  --mimic-tool-present-csv logs/mimic/analysis/tool_present_subgroup_metrics_only.csv \
  --out-dir analysis/paper_figures
```

This consumes the five driver-LLM `fairness_posthoc` outputs for both datasets and writes only:

- `figures/paper_figure2_tool_exposure_bias.png`
- `figures/paper_figure3_tool_transition_bias.png`
- `paper_figure2_tool_exposure_data.csv`
- `paper_figure3_tool_transition_data.csv`
- `paper_figure_manifest.json`

`fairness_posthoc.py --paper-figures-only` is only a single-log diagnostic and does not reproduce the full paper figure, because the paper figures aggregate across five driver LLMs and both datasets.

## Main Outputs

General outputs:

- `per_question_features.csv`: joined per-question table with predictions, subgroup attributes, tool traces, and text features.
- `group_performance.csv`: subgroup end-to-end accuracy.
- `group_performance_intersectional.csv`: gender-by-age subgroup accuracy.
- `summary_report.md`: compact summary of coverage, subgroup performance, and process-level disparities.

Paper decomposition outputs:

- `lens_agentic_conditional_tool_utility.csv`
- `lens_agentic_conditional_tool_utility_gap.csv`
- `lens_agentic_transition_rates_by_group.csv`
- `lens_agentic_transition_divergence.csv`
- `lens_llm_reasoning_bias.csv`
- `lens_llm_reasoning_bias_by_group.csv`

Optional supporting outputs:

- `lens_inherited_tool_bias.csv`: tool failure and tool-use associations.
- `lens_agentic_tool_selection_bias.csv`: marginal tool-use gaps.
- `lens_agentic_tool_usage_gap_ci.csv`: bootstrap confidence intervals for tool-use gaps.
- `llm_reasoning_judge_scores.csv`: raw per-question judge scores when `--enable-llm-judge` is used.
