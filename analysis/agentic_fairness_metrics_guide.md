# Agentic Subgroup Fairness Metrics Guide

This guide documents the metrics and outputs used for the "Agentic bias via tool selection" lens in `analysis/fairness_posthoc.py`.

Scope:
- Tool selection fairness across demographic subgroups.
- Subgroup-conditional tool utility fairness.
- Uncertainty-aware usage-gap reporting via bootstrap confidence intervals.


## Core Fairness Question
Does the agent choose different tools and tool paths across subgroups, and do those differences align with subgroup performance disparities?

## Data Inputs
- `per_question_features.csv` generated by `analysis/fairness_posthoc.py`.
- Sensitive attributes used by default:
- `gender_norm`
- `age_group`

The script removes unknown subgroup labels for subgroup disparity analyses where specified.

## Metric Family A: Tool Selection Gap (Existing Lens-2 Baseline)

Output:
- `lens_agentic_tool_selection_bias.csv`

Columns:
- `attribute`: Sensitive attribute (`gender_norm` or `age_group`).
- `tool`: Tool name.
- `usage_gap`: Max subgroup usage rate minus min subgroup usage rate for this tool.

What fairness dimension it measures:
- Exposure parity of each tool across subgroups.
- Detects if planner policy uses tools differently by subgroup.

How to interpret:
- Higher `usage_gap` means larger subgroup imbalance in tool exposure.
- `usage_gap ~= 0` suggests exposure parity for that tool.
- This metric does not say whether the disparity is harmful or beneficial.


## Metric Family: Tool-Order Transition Disparity

Outputs:
- `lens_agentic_transition_rates_by_group.csv`
- `lens_agentic_transition_divergence.csv`
- Figure: `figures/lens_agentic_transition_divergence_<attribute>.png`

Definition:
- Convert each tool sequence into transitions:
- `START -> first_tool`
- `tool_i -> tool_{i+1}`
- Build transition distributions per subgroup.
- Compare subgroup distributions using Jensen-Shannon divergence (JSD).

Columns (`lens_agentic_transition_rates_by_group.csv`):
- `attribute`
- `group`
- `transition`: e.g., `START->XRayVQATool`, `XRayVQATool->ChestXRayClassifierTool`.
- `count`
- `rate`: Transition probability within subgroup.
- `n_questions`
- `n_transitions`

Columns (`lens_agentic_transition_divergence.csv`):
- `attribute`
- `group_a`
- `group_b`
- `js_divergence`: Symmetric divergence between transition distributions.
- `n_questions_a`
- `n_questions_b`

Fairness meaning:
- Captures order-level planning differences, not just marginal tool-use rates.

How to interpret:
- JSD near 0: subgroup tool-order policies are similar.
- Higher JSD: subgroup trajectory patterns differ materially.
- Use `transition_rates_by_group` to locate which transitions drive high JSD.


## Metric Family: Usage Gap with Uncertainty (New)

Output:
- `lens_agentic_tool_usage_gap_ci.csv`
- Figure: `figures/lens_agentic_tool_usage_gap_ci_<attribute>.png`

Definition:
- For each tool:
- Find highest-usage subgroup and lowest-usage subgroup.
- Report usage-rate gap and bootstrap CI for the difference.

Columns:
- `attribute`
- `tool`
- `high_group`
- `low_group`
- `high_group_rate`
- `low_group_rate`
- `gap`
- `ci_low`
- `ci_high`
- `n_high`
- `n_low`
- `support_ok`

Fairness meaning:
- Adds statistical stability information to usage disparity.

How to interpret:
- If CI excludes 0, exposure disparity is more stable.
- If CI overlaps 0, disparity may be weak/noisy at current sample size.

## Support Thresholds Used in Script
- `MIN_GROUP_SUPPORT = 10`
- `MIN_BUCKET_SUPPORT = 8`

These thresholds reduce unstable subgroup inferences. Rows failing support checks are retained with missing effect estimates where relevant.

## Suggested Analysis Workflow
1. Check exposure disparity:
- `lens_agentic_tool_selection_bias.csv`
- `lens_agentic_tool_usage_gap_ci.csv`

2. Check order-level disparity:
- `lens_agentic_transition_divergence.csv`
- `lens_agentic_transition_rates_by_group.csv`

3. Check path-attributable performance disparity:
- `lens_agentic_counterfactual_policy_summary.csv`
- `lens_agentic_path_conditioned_gap.csv`

4. Check tool-level unequal utility:
- `lens_agentic_conditional_tool_utility_gap.csv`
- `lens_agentic_conditional_tool_utility.csv`

5. Prioritize mitigations:
- Tools with high exposure gap + negative subgroup utility + high support.
- Transition edges with high subgroup skew and large subgroup performance differences.

## Practical Interpretation Patterns

Pattern 1:
- High usage gap
- High transition JSD
- Large positive `gap_reduction`
Interpretation:
- Planner policy likely contributes materially to subgroup disparity.

Pattern 2:
- Low usage gap
- Low transition JSD
- Large residual path-standardized gap
Interpretation:
- Disparity likely driven more by tool reliability differences or reasoning behavior than by tool selection policy.

Pattern 3:
- Large conditional utility gap for a specific tool
- Tool heavily used in lower-performing subgroup
Interpretation:
- Candidate for subgroup-aware reliability gating, fallback routing, or recalibration.

## Limitations
- Observational analysis can be confounded by unmeasured case severity/content differences.
- CIs here are bootstrap CIs over observed samples, not full causal uncertainty intervals.
- Small subgroups may still produce unstable estimates despite support thresholds.

## Recommended Next Validation Step
- Run targeted counterfactual replay:
- Keep question/case fixed.
- Force matched tool policies across groups.
- Measure whether subgroup gaps shrink.

This is the strongest follow-up test for whether agentic tool selection itself is a principal fairness driver.
