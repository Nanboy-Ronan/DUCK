# Agentic Fairness Metrics Guide

This guide maps `analysis/fairness_posthoc.py` outputs to the DUCX paper's process-level fairness decomposition.

DUCX argues that final-answer fairness alone is insufficient for tool-using medical agents. A demographic gap can arise from the final model response, but it can also arise earlier in the agent trajectory: which tools are used, how the agent moves between tools, and how the driver LLM synthesizes tool outputs.

## Inputs

Primary input:

- `per_question_features.csv`, generated by [fairness_posthoc.py](fairness_posthoc.py).

Sensitive attributes used by default:

- `gender_norm`
- `age_group`

Unknown subgroup labels are excluded from subgroup disparity calculations where appropriate. For ChestAgentBench, age is represented as the paper's binary `young` / `old` grouping; for MIMIC-FairnessVQA, age is binarized at 60.

## End-to-End Context

Before interpreting process-level outputs, inspect final prediction performance:

- `group_performance.csv`
- `group_performance_intersectional.csv`

These files show whether the final agent answers differ across gender, age, or intersectional subgroups. The paper then asks whether such gaps are reflected in intermediate agent behavior.

## Lens 1: Tool Exposure Bias

Paper question:

When a given tool appears in the trajectory, does the benefit or harm associated with that tool differ across demographic groups?

Primary files:

- `lens_agentic_conditional_tool_utility.csv`
- `lens_agentic_conditional_tool_utility_gap.csv`

Interpretation:

- `lens_agentic_conditional_tool_utility.csv` reports subgroup performance conditioned on tool presence.
- `lens_agentic_conditional_tool_utility_gap.csv` reports the largest subgroup utility gaps for each tool.
- A large gap means that the same tool exposure is associated with different downstream utility across subgroups.

This corresponds to the paper's **tool exposure bias**: subgroup utility gaps conditioned on whether a tool is present in the agent trajectory.

Supporting file:

- `lens_agentic_tool_selection_bias.csv`

This supporting file reports marginal tool-use gaps. It answers a related but narrower question: whether subgroups are exposed to a tool at different rates. In the paper framing, rate differences help contextualize exposure bias, but the primary exposure metric is utility conditioned on tool presence.

## Lens 2: Tool Transition Bias

Paper question:

Does the agent route different demographic groups through different tool chains?

Primary files:

- `lens_agentic_transition_rates_by_group.csv`
- `lens_agentic_transition_divergence.csv`

Definitions:

- Each trajectory is converted into transitions such as `START->XRayVQATool` and `XRayVQATool->ChestXRayClassifierTool`.
- For each subgroup, the script estimates transition rates over tool paths.
- `lens_agentic_transition_divergence.csv` compares subgroup transition distributions.

Interpretation:

- Similar transition distributions indicate similar routing behavior.
- Larger transition divergence indicates that the agent follows different process paths for different groups.
- Use `lens_agentic_transition_rates_by_group.csv` to identify which transitions explain a high divergence value.

This corresponds to the paper's **tool transition bias**: subgroup differences in tool-routing patterns.

## Lens 3: Model Reasoning Bias

Paper question:

After tools have been used, does the driver LLM synthesize answers differently across demographic groups?

Primary files:

- `lens_llm_reasoning_bias.csv`
- `lens_llm_reasoning_bias_by_group.csv`

Features:

- `reasoning_quality_score`: external LLM-judge score, available when `--enable-llm-judge` is used.
- `hedge_count`: uncertainty language such as "may", "possibly", or "likely".
- `demographic_terms`: explicit demographic framing such as age or gender terms.

Additional lexical features such as `certainty_count`, `refusal_flag`, and `inconsistency_markers` are included by the code as supporting diagnostics, but the paper's core reasoning-bias interpretation centers on reasoning quality, uncertainty expression, and demographic framing.

Interpretation:

- A large `abs_gap` in `reasoning_quality_score` suggests subgroup differences in final synthesis quality.
- A large hedging gap suggests demographic differences in uncertainty expression.
- A large demographic-term gap suggests demographic framing differences in the final answer text.

This corresponds to the paper's **model reasoning bias**: subgroup differences in synthesis behavior.

## Recommended Paper-Aligned Workflow

1. Check final answer disparities:

   `group_performance.csv`, `group_performance_intersectional.csv`

2. Inspect tool exposure bias:

   `lens_agentic_conditional_tool_utility_gap.csv`

3. Inspect tool transition bias:

   `lens_agentic_transition_divergence.csv`, then drill into `lens_agentic_transition_rates_by_group.csv`

4. Inspect reasoning bias:

   `lens_llm_reasoning_bias.csv`, then drill into `lens_llm_reasoning_bias_by_group.csv`

5. Compare process-level disparities with final-answer gaps:

   The paper's key point is that intermediate disparities can appear even when end-to-end metrics alone do not explain where unfairness entered the system.

## Strict Figure Reproduction

Use [generate_paper_figures.py](generate_paper_figures.py) when the goal is to reproduce the paper figures. It consumes the five driver-LLM `fairness_posthoc` outputs for CheXAgentBench and MIMIC-FairnessVQA, writes the backing CSV data for the paper's tool exposure and transition figures, and leaves only paper-figure artifacts in the output directory.

`fairness_posthoc.py --paper-figures-only` is only a single-log diagnostic. It should not be used as the full paper-figure reproduction command because Fig. 2 and Fig. 3 aggregate across five driver LLMs and both datasets.

## Support And Stability

The script uses support thresholds to avoid over-interpreting tiny subgroups or rare trajectory buckets:

- `MIN_GROUP_SUPPORT = 10`
- `MIN_BUCKET_SUPPORT = 8`

Rows that do not meet support requirements may be retained but should be treated as unstable.

`lens_agentic_tool_usage_gap_ci.csv` provides bootstrap confidence intervals for marginal tool-use gaps. This is useful supporting context, but it is not the central paper definition of tool exposure bias.

## Out Of Scope For Paper Reproduction

The following outputs may be useful for development or follow-up work, but they should not be presented as additional paper claims unless explicitly described:

- `lens_agentic_counterfactual_policy_eval.csv`
- `lens_agentic_counterfactual_policy_summary.csv`
- `lens_modular_mitigation_opportunities.csv`
- `lens_traceable_reasoning_audit.csv`
- `lens_interaction_bias_summary.csv`

They are retained in the codebase for transparency and future analysis, while the public paper reproduction path should focus on tool exposure, tool transition, and model reasoning bias.

## Limitations

- The decomposition is observational over logged agent trajectories.
- Tool exposure and transition disparities can reflect case mix, data coverage, tool reliability, or planner behavior.
- LLM-judge scores depend on the selected judge model and should be reported with the judge configuration.
- Small subgroups and rare tool paths require cautious interpretation.
